# Database Engineering Standards

## Purpose

Design scalable relational databases.

---

## Rules

- Normalize appropriately.
- Use foreign keys.
- Add indexes where useful.
- Avoid duplicate data.

---

## Naming

- snake_case
- id as primary key
- table_id as foreign key

---

## Performance

- Optimize joins.
- Index search columns.
- Avoid unnecessary queries.

---

## Checklist

- Normalized
- Indexed
- Secure
- Scalable

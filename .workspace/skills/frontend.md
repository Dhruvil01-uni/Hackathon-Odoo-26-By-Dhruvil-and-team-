# Frontend Engineering Standards

## Purpose

Define standards for building modern frontend applications.

---

## Project Structure

- components/
- pages/
- layouts/
- hooks/
- services/
- utils/
- types/

---

## Component Rules

- One responsibility per component.
- Prefer composition over duplication.
- Keep props minimal.
- Reuse before creating.

---

## State Management

- Local state first.
- Lift state only when necessary.
- Keep business logic outside UI.

---

## Styling

- Tailwind CSS preferred.
- Avoid inline styles.
- Maintain consistent spacing.
- Mobile-first design.

---

## Performance

- Lazy load pages.
- Memoize expensive operations.
- Optimize rendering.

---

## Checklist

- Responsive
- Accessible
- Reusable
- Type Safe
- Optimized

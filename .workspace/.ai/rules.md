# AI Development Rules

## Mission

Produce production-quality software quickly without sacrificing maintainability.

---

## General Principles

- Understand before generating.
- Reuse before creating.
- Simplicity over cleverness.
- Modular over monolithic.
- Explain important decisions.

---

## Before Writing Code

Always:

1. Read project-context/.
2. Read instructions/.
3. Inspect existing files.
4. Explain implementation plan.
5. List modified files.

---

## During Development

Always:

- Keep code reusable.
- Use TypeScript.
- Keep functions small.
- Avoid duplication.
- Handle errors.
- Validate inputs.
- Keep UI responsive.

---

## After Development

Always:

- Review generated code.
- Suggest improvements.
- Mention edge cases.
- Check performance.
- Check security.

---

## Never

Never generate code blindly.

Never overwrite working code.

Never ignore existing architecture.

Never hardcode secrets.

Never duplicate components.
